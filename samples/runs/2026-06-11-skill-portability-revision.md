# 2026-06-11 Skill 可移植性修订记录

## 触发问题

用户指出：

- 医学视频总结只有事实核查，没有展开反对观点。
- skill 不应只依赖 Codex 的强理解和补全能力，而应能迁移到其他 Agent 系统中正常工作。

## 问题判断

原 skill 的问题不在于完全没有 `Counter`，而在于触发条件太窄：

- 只有 `argument` 模式明确要求 SFC。
- `information` 模式没有要求“只要包含建议、因果、风险、医学/科学/产品/财经等 claim-bearing 内容，就必须追加反方观点和边界条件”。
- 对弱 Agent 来说，它很可能把健康科普、产品推荐、资讯点评都当作普通信息摘要，从而漏掉反方观点。

## 已做修订

### 1. 强化 claim-bearing override

新增硬规则：

只要视频包含以下内容，即使分类为信息类，也必须加 `Fact Check` 和 `Counter / Boundaries`：

- 建议、因果、风险、预测、价值判断
- 医学、健康、财经、法律、安全、科学、统计
- 产品/方法推荐或排行
- “必须、最好、唯一、一定、proven”等强表述

### 2. 补充 Counter 质量要求

反方观点必须至少包括：

- 最强反向解释
- 边界条件
- 反例
- 缺失证据
- 盲从可能伤害
- 什么证据会改变结论

### 3. 加入弱 Agent 执行循环

新增显式工作流：

1. 分类输入和输出模式
2. 获取或请求文本证据
3. 规范化转写
4. 抽取 claims
5. 判断是否必须 fact check / counter
6. 套用输出模板
7. 标记置信度、缺口、下一步

### 4. 补充无工具降级路径

当没有 Codex、本地脚本、浏览器或联网能力时：

- 请求用户提供字幕/转写/时间戳摘录
- 只有标题简介时，只做候选级判断
- 无法核查时标记 `needs verification`
- 禁止伪造引用和核查结论

### 5. 增加跨 Agent 可复制提示词

新增：

- `skills/bilibili-video-analysis/PORTABLE_AGENT_PROMPT.md`

用于复制到其他 Agent 系统中。

## 对医学视频总结的修正方向

以后类似“睡眠/健康科普”输出必须包含：

- 视频核心主张
- 视频证据链
- 事实核查表
- 反方观点与边界条件
- 哪些人不能直接照做
- 哪些说法需要回看或外部来源核查

示例：

| 视频主张 | 反方/边界 |
|---|---|
| 晚睡晚起也算熬夜 | 如果总睡眠足够、作息长期稳定、白天光照和社会节律安排合理，风险可能低于“短睡 + 高压”的情况 |
| 少睡会提高死亡风险 | 观察性研究难完全排除反向因果和混杂因素，例如疾病导致短睡，也导致死亡风险升高 |
| 熬夜后补 B 族维生素有助恢复 | 若无缺乏证据，补充剂效果有限，不能替代睡眠恢复；过度强调补剂可能误导 |

## 验收结论

修订后 skill 更适合迁移：

- Codex 可继续用本地脚本自动跑链路。
- 其他 Agent 可按 portable prompt 要求用户提供 transcript。
- 弱 Agent 也有硬性质量门槛，不应漏掉 Counter / Boundaries。
