# 2026-06-11 视觉证据低成本策略修订

## 触发问题

用户指出：B站视频经常依赖画面信息，字幕/ASR 可能不足。希望 Agent 在视频依赖画面时主动尝试截屏，但不要一次请求消耗太多 Agent 调用额度。

## 修订原则

- 不默认全片抽帧。
- 不把“看过 ASR”当作完整理解。
- 只在视觉证据可能改变总结、事实核查、反方观点或操作步骤时截屏。
- 默认走最小成本路径。

## 新规则

### 何时必须考虑视觉证据

出现以下情况时，必须判断是否需要截图：

- 工具教程、软件操作、代码演示、UI 流程
- 游戏/产品发布、排行榜、参数对比、清单
- PPT/课程幻灯片、表格、图表、地图、医学图示
- 视频多次出现“如图、这个表、画面里、左边、右边、这里”
- ASR 对关键专名、数字、列表、产品名不确定

### 截图调用梯度

1. 不截图：纯讲座/访谈且转写足够。
2. 最小截图：3-5 张关键帧。
3. 聚焦截图：6-12 张，仅在第一轮发现图表/列表/UI/代码密集时。
4. 重视觉分析：需要用户确认后再做，例如全片抽帧、逐帧 OCR、长时间浏览器观看。

### 选帧策略

优先用转写时间戳：

- 开头标题卡
- 主要 claim 切换点
- ASR 不确定的名字/数字/列表附近
- “如图/这个表/看这里”附近
- 结尾总结画面

无时间戳时才用 `5% / 25% / 50% / 75% / 95%`。
